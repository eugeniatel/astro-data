  <action>
Create vite.config.ts at project root with the following exact content. The optimizeDeps.exclude is CRITICAL -- without it, Vite rewrites the import.meta.url path inside swisseph-wasm and breaks the swisseph.data file location. The assetsInclude ensures the .data file is served as a static asset:

```typescript
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig({
  optimizeDeps: {
    exclude: ['swisseph-wasm'],
  },
  server: {
    fs: {
      allow: ['..'],
    },
  },
  assetsInclude: ['**/*.data'],
});
```

Create vitest.config.ts at project root:

```typescript
// vitest.config.ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    environment: 'jsdom',
    globals: true,
    include: ['src/**/*.test.ts'],
    reporter: 'verbose',
  },
});
```

Note: vitest uses jsdom environment because the app targets browser APIs (localStorage, fetch). The globals:true setting enables describe/it/expect without imports in test files.

Verify the dev server starts (check only, do not leave running):


Run vitest with no tests to confirm it initializes:
